This is the first release of the model, and it's built to be improved on. A linear model was the right starting point for a transparent baseline, but the roadmap includes testing tree-based approaches like Random Forest and Gradient Boosting, and adding features such as proximity to main roads, floor count, and neighborhood-level amenity scores. The goal is a tighter gap between MAE and RMSE, and a higher R² without losing the ability to explain why the model landed on a given price.

The Project

How it's built

This tool predicts residential property prices in Lahore using a supervised regression pipeline built with Scikit-learn and served through FastAPI. Location, area, bedrooms, bathrooms, kitchens, built year, store rooms, and servant quarters are engineered into features, with location encoded numerically alongside the physical property attributes. The model trains on the bulk of the 18,000+ listings and is scored against a held-out set it never saw during training, using R², MAE, and RMSE together rather than a single accuracy figure, since price prediction is a continuous target, not a classification problem.
